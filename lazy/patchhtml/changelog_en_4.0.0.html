4.0.3 Important Enhancements/Notes (8/30/2022)

Swap Hands modifiers

One-handed weapons can now be wielded two-handed and two-handed weapons the other way around. All modifiers are included. You are allowed to swap hands once every time it is your turn in combat. Using a shield with a spear is now possible!

New item "information"

The new item type "information" is now available. It can start a roll check to define how many information an user knows about a specific topic. The player or GM (configurable in the settings) receives a whispered message according to the result of the roll and can then process the information as he sees fit (e.g. tell the other players). We updated all the creatures to include those informations where necessary. Later on we will also update all plants.

Artifact creation

The core module now brings an artifact creation helper to the player menu. It assists in creating magical artifacts by showing all roll modifiers, costs and other important parts of the process. We will probably add more helpers for complex rule elements like this in the future.

Active Effects for weapons and other items

Active effects can now modify other items. You can now create e.g. a ring which modifies your preferred weapon and increases the attack mod value.
This will come in handy for special abilites for traditional artifact items and the like. We will add those to the compendiae at a later time. Fields that have been modified by such an active effect can not be changed manually anymore until the effect is gone!
Example key: "@meleeweapon.Dagger.system.atmod.value, mode: add, value 2 - would increase the attack value of each item named dagger of type meleeweapon by 2

Enhancements

  • Active Effects of special abilites, advantages and disadvantages are now applied according to their level x times
  • The master menu has now a button to drag all tracked actors at the same time to the canvas
  • The library has now a loading bar, so you know what's going on
  • You can now drag actors from the master menu directly to the canvas
  • You can now disable the combat regeneration of creatures (e.g. if you hit the werewolf with a silver sword)
  • The stituational modifiers should have a more helpful tooltip now
  • Traditional artifact special abilities now have their respective artifact as icon so they can be found easier in the library
  • The TDE journal sheet style can now for the most part be deactivated by switching the journal sheet
  • A few tours have been added
  • The tooltips have been migrated to the v10 default style
  • Regeneration rolls can now be selective (e.g. only AE, LE)
  • Actors now have a method to detect the creature class which can be used to define e.g. blood colors for the module splatter. The core module contains a predefined macro to set those colors (thanks to VTTom). You can modify it to your likings.
  • We added a few more map note icons
  • An automatic self-control check request is now posted to the chat whenever an actor receives 4 level of pain, so he can try to avoid becoming incapacitated (this can be disabled in the options)
  • TDE is now bundled and compressed which should improve loading time (a bit experimental)
  • Enabling or disabling access of books for the players now also shows or hides the connected compendiae
  • Automatic renaming of tokens dropped to the canvas can now be done without numbering
  • The lighthelper in the core module has now all light settings available. Moreover the token player HUD can now be customized with own light configurations.
  • GM arbitrariness now also works for hitzones
  • The "did you know message" now fades after 8s (instead of 4) and the fade can be stopped entirely. After the stop you can request more messages. Just hit the right button.
  • Ini and movement modifiers for zone armor should be calculated properly now (thanks to Qowy)
  • The creature type detection now also works for the "npc" and "character" sheet (instead of "creature" only). Keep in mind that the species field is readonly as the species should only be dragged to the character to ensure all base attributes are set properly. To make the detection work, you need to create an active effect which overrides the species name (this is included in the lykanthropie and wercreature conditions)
  • Status effects of actors can now be transferred to other actors via drag and drop (drag icon)

Bugfixes

  • Parry rolls should now properly show the token name instead of the actor name
  • Fixed calculation of wercreature damage modifiers
  • Fixed level depiction of general special abilities for the character sheet
  • Auto pan to token in combat shouldn't occur so excessive anymore
